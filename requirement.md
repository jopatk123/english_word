# 个人词根背单词工具 - 需求文档
## 1. 项目概述
### 1.1 项目背景
为满足个人高效背单词的需求，基于“词根记忆法”核心逻辑，开发一款私有化部署的背单词工具，支持手动管理词根、单词、例句，数据本地存储，支持跨设备访问，整体实现轻量化、一体化部署。

### 1.2 核心目标
- 以词根为核心层级，实现词根、单词、例句的增删改查（CRUD）管理；
- 数据持久化存储，支持私有化部署，无外部依赖；
- 操作界面简洁，满足个人日常背单词记录、查阅需求；
- 一体化部署，可通过Docker快速启动，跨环境运行。

## 2. 功能需求
### 2.1 核心功能模块
| 模块       | 功能点                                                                 |
|------------|------------------------------------------------------------------------|
| 词根管理   | 1. 添加词根（必填：词根、核心含义；可选：备注）<br>2. 编辑词根信息<br>3. 删除词根（级联删除关联的单词、例句）<br>4. 查看词根列表及关联单词数量 |
| 单词管理   | 1. 为指定词根添加单词（必填：单词、含义；可选：音标、备注）<br>2. 编辑单词信息<br>3. 删除单词（级联删除关联的例句）<br>4. 查看词根下的单词列表 |
| 例句管理   | 1. 为指定单词添加例句（必填：例句原文、翻译；可选：备注，如“高频考点”）<br>2. 编辑例句信息<br>3. 删除例句<br>4. 查看单词下的例句列表 |
| 基础搜索   | 1. 按词根模糊搜索<br>2. 按单词模糊搜索                                 |
| 数据管理   | 1. 数据本地持久化存储<br>2. 支持手动备份数据库文件（可选）             |

### 2.2 非功能需求
- **易用性**：界面简洁，操作流程无冗余，支持鼠标/键盘快速操作；
- **适配性**：前端响应式设计，适配手机移动端访问；
- **性能**：单用户场景下，所有操作响应时间≤1秒；
- **兼容性**：前端兼容主流浏览器（Chrome、Edge、Firefox）；
- **稳定性**：Docker部署后，容器异常退出可自动重启，数据不丢失。

## 3. 技术要求
### 3.1 技术栈选型（强制）
| 技术层面   | 选型要求                                                                 |
|------------|--------------------------------------------------------------------------|
| 后端       | Node.js + Express<br>- 需使用ES6+语法，代码模块化拆分<br>- 接口遵循RESTful规范<br>- 统一返回格式：{code: 200/500, data: {}, msg: ""} |
| 前端       | Vue3 + Vite<br>- 采用组合式API（Setup语法糖）开发<br>- 组件化拆分（词根、单词、例句组件解耦）<br>- 无需复杂UI框架，优先使用原生CSS或轻量UI库（如Element Plus） |
| 数据库     | SQLite + ORM<br>- ORM框架：Sequelize（需配置模型关联、级联删除）<br>- 数据库文件路径固定为容器内 `/app/data/words.db` |
| 部署       | Docker + Docker Compose<br>- 多阶段构建Docker镜像，减小镜像体积<br>- 配置卷挂载保证数据持久化<br>- 容器暴露端口为3000 |

### 3.2 技术约束
1. **数据库设计**：
   - 必须设计3张核心表：词根表（roots）、单词表（words）、例句表（examples）；
   - 表之间通过外键关联，删除词根/单词时需触发级联删除；
   - 核心表字段需包含：创建时间（create_time），可选包含更新时间（update_time）。
2. **后端接口**：
   - 所有接口需处理参数校验，返回清晰的错误提示；
   - 接口路径规范：`/api/roots`（词根）、`/api/words`（单词）、`/api/examples`（例句）；
   - 支持跨域请求（CORS）配置。
3. **前端开发**：
   - 前端静态资源由Express托管，访问根路径（/）直接加载前端页面；
   - 前端路由使用Vue Router，需处理刷新页面404问题；
   - 无登录鉴权模块（个人工具无需）。
4. **Docker部署**：
   - Dockerfile需基于`node:18-alpine`镜像构建；
   - docker-compose.yml需配置：端口映射（3010:3010）、数据卷挂载（宿主机./data → 容器/app/data）、自动重启（restart: always）；
   - 镜像构建完成后，需保证执行`docker-compose up -d`即可一键启动服务。

## 4. 部署与运行要求
### 4.1 部署环境
- 支持Linux/macOS/Windows（需安装Docker Desktop）；
- 宿主机无需预装Node.js、Vue等环境，完全基于Docker隔离运行。

### 4.2 运行流程
1. 克隆项目后，执行`docker-compose build`构建镜像；
2. 执行`docker-compose up -d`启动服务；
3. 访问`http://localhost:3010`即可使用工具；
4. 数据持久化：宿主机`./data`目录存放SQLite数据库文件，手动备份该目录即可。
